Inhalt der Vorlesung

Die Vorlesung „Webprogrammierung“ des dritten Semesters bildet zusammen mit „Verteilte Systeme“ im vierten Semester das Modul „Entwicklung verteilter Systeme“, durch welches die Programmierschiene des Studiums fortgeführt wird. Aufbauend auf den in „Programmierung I“ und „Programmierung II“ gelehrten Grundlagen verlagert sich der Schwerpunkt auf die Entwicklung verteilter Anwendungen, bei denen sich die Anwendungslogik auf mehrere Maschinen wie zum Beispiel Client und Server verteilt. Hierunter fallen Webanwendungen, Cloud-Anwendungen, IoT-Anwendungen, mobile Applikationen und viele mehr. Das Modul teilt sich inhaltlich daher in die Grundlagen der Webentwicklung im dritten Semester und die Entwicklung komplexer Backendstrukturen im vierten Semester auf. Da die Grenzen zwischen den Anwendungstypen aus technischer Sicht fließend verlaufen, dient dies gleichzeitig als Grundlage für die Wahlmodule „Entwicklung mobiler Applikationen“ und „Hardwarenahe Programmierung für das Internet of Things“. Konkret werden im dritten Semester folgende Inhalte behandelt:

  • Die geschichtliche Entwicklung des Internets und des World Wide Web
  • Das Zusammenspiel der Web-Technologien HTTP, HTML, CSS und JavaScript
  • Die inhaltliche Strukturierung von Webseiten und webbasierter UIs mit HTML
  • Die optische Aufbereitung und Gestaltung von Webinhalten mit CSS
  • Responsive Webdesign nach dem Mobile-First-Ansatz
  • Die Entwicklung clientseitiger Anwendungslogik mit JavaScript
  • Das Document Object Model als Schnittstelle zwischen HTML, CSS und JavaScript
  • Die Strukturierung und Entwicklung skalierbarer Single Page Apps mit JavaScript

Didaktisches Modell

Das didaktische Modell der Vorlesung entspricht weitgehend dem Inverted Classroom Model, wandelt dieses jedoch zu Gunsten einer besseren Inhaltsvermittlung und Vertiefung an einigen Punkten ab. So werden die Inhalte nicht durch aufgezeichnete Videos oder Web Based Trainings, bei denen mehr oder weniger die selben Folien wie in der klassischen Präsenzlehre verwertet werden, sondern durch ein speziell aufbereitetes Onlinelehrbuch vermittelt. Das Onlinelehrbuch besteht hierzu aus schriftlich ausformulierten Inhalten, Visualisierungen, interaktiven Elementen, Screencasts, Quizes und Übungsaufgaben, um die Inhalte bestmöglich zu präsentieren. Auf diese Weise soll die aktive Auseinandersetzung mit den Lerninhalten gefördert werden, anstatt diese zum passiven Konsum anzubieten. Die Kapitel sind daher wie folgt gegliedert:

  1. Selbststudium des Onlinelehrbuchs
  2. Eine oder mehrere Theorievorlesungen
  3. Prüfungsrelevante Lernkontrolle
  4. Eine oder mehrere Übungsstunden

Im Anschluss an das Selbststudium können die Studierenden die Lerninhalte anhand eines einfachen Feedbackformulars reflektieren sowie Fragen für die nächste Präsenzvorlesung stellen. Dies findet zusammen mit dem benoteten Online-Assessment als Lernkontrolle derzeit noch in Moodle statt, wird in künftig aber ebenfalls innerhalb des Onlinelehrbuchs angeboten, um die Teilnahmequote zu erhöhen.
